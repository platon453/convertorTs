  План по расширению проекта

  Фаза 1: Добавление конвертации USD → RUB

   * Backend (наш сервер):
       1. Модификация API: Мы изменим эндпоинт /api/rate, чтобы он мог принимать название валюты (например, /api/rate/EUR или /api/rate/USD).
       2. Обновление сервиса: Логика в rateService.ts будет обновлена для динамической загрузки и кэширования курсов для разных валютных пар (не только EUR-RUB, но и USD-RUB).

   * Frontend (сайт):
       1. Обновление интерфейса: Добавим на страницу элементы для выбора валюты (EUR или USD), как решим в вопросе №2.
       2. Обновление логики: Запрос к нашему API будет изменен так, чтобы он запрашивал курс для выбранной пользователем валюты.

  Фаза 2: Отображение истории курса

  На этом этапе мы добавим визуальный элемент, который покажет динамику курса за последние несколько дней, чтобы пользователь мог принять решение об обмене.

   * Backend:
       1. Новый эндпоинт: Создадим новый API-эндпоинт, например, /api/history/EUR, который будет возвращать данные о курсе за последние N дней.
       2. Новый сервис: Реализуем логику, которая будет запрашивать исторические данные у внешнего API.

   * Frontend:
       1. Новый компонент: Выделим на странице место для графика.
       2. Запрос истории: Напишем функцию для получения исторических данных с нашего нового эндпоинта.
       3. Визуализация: Мое предложение — вместо обычного графика нарисовать его в виде ретро ASCII-диаграммы прямо в терминальном окне. Это будет выглядеть очень стильно, необычно и полностью впишется в
          хакерскую тематику сайта.
